# Анализатор лендингов + Яндекс Метрика + Telegram-бот

## Что делает проект

Собирает статистику конкретного лендинга из Яндекс Метрики (визиты, просмотры,
пользователи, отказы, время на сайте, источники трафика), анализирует её по
простым правилам и отправляет отчёт (текст + диаграмма) в Telegram.

## Архитектура

Один главный агент (orchestrator) координирует субагентов:

- **metrika_agent** — получает данные из Яндекс Метрики (Stat API), умеет
  фильтровать по конкретному URL лендинга и задавать период (`7daysAgo`,
  `30daysAgo`).
- **analyzer_agent** — проверяет метрики по пороговым значениям (высокий
  процент отказов, короткое время на сайте) и формирует текстовые выводы.
- **report_agent** — собирает текстовый отчёт на русском языке.
- **chart_agent** — строит диаграмму источников трафика (matplotlib).
- **orchestrator** — вызывает агентов по очереди и возвращает `(отчёт, график)`.

## Точки входа

- `src/main.py` — одноразовый запуск: собрать отчёт за неделю по лендингу из
  `LANDING_PATH` и отправить в Telegram.
- `src/bot.py` — интерактивный Telegram-бот с командой `/report [лендинг]
  [week|month]`. Запускается вручную (`python src/bot.py`), работает пока
  открыт терминал (long polling).

## Настройка

Все секреты — в `.env` в корне проекта (не попадает в git). Шаблон без
секретов — `config/.env.example`.

Нужные переменные:
- `YANDEX_METRIKA_TOKEN` — OAuth-токен с правом чтения статистики
- `YANDEX_METRIKA_COUNTER_ID` — ID счётчика Метрики
- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather
- `TELEGRAM_CHAT_ID` — ID чата, куда слать отчёты
- `LANDING_PATH` — URL лендинга для анализа (например `zerocoder.ru/v1-claude-code`)

## Возможные следующие шаги

- Автоматический еженедельный запуск (Планировщик задач Windows / cron)
- Сравнение метрик с предыдущим периодом
- Дополнительные графики (динамика по дням, конверсии по целям)
