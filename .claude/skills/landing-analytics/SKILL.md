# Landing Analytics — гид по проекту

## Что это за проект

Python-скрипт + Telegram-бот, который берёт статистику лендинга из Яндекс
Метрики (Stat API), анализирует её по простым правилам и отправляет отчёт
(текст + диаграмма источников трафика) в Telegram.

## Структура репозитория

```
landing-analytics/
├── src/
│   ├── main.py          ← одноразовый запуск: отчёт по LANDING_PATH из .env
│   ├── bot.py             ← интерактивный Telegram-бот (/report, реакция на ссылки)
│   ├── config.py          ← читает переменные окружения из .env
│   ├── telegram_bot.py    ← send_message() / send_photo() через Bot API
│   └── agents/
│       ├── orchestrator.py    ← координирует субагентов, run_weekly_report()
│       ├── metrika_agent.py   ← запросы к Яндекс Метрике (Stat API)
│       ├── analyzer_agent.py  ← пороги (bounce_rate, avg_duration) -> insights
│       ├── report_agent.py    ← сборка текста отчёта (период, источники, UTM)
│       └── chart_agent.py     ← диаграмма источников трафика (matplotlib)
├── config/.env.example     ← шаблон переменных окружения (без секретов)
├── data/example_report.txt ← пример отчёта
├── docs/project.md          ← подробное описание архитектуры
├── tests/                    ← заготовка под тесты
├── .env                       ← реальные токены и ID (НЕ в git, в .gitignore)
├── requirements.txt
└── README.md
```

## Виртуальное окружение

Python устанавливается как `py` (не `python`). Окружение — `venv/` в корне.
Все команды запускать через:

```
./venv/Scripts/python.exe <скрипт>
```

Если `venv/` отсутствует:
```
py -m venv venv
./venv/Scripts/python.exe -m pip install -r requirements.txt
```

## Как запустить

Разовый отчёт по лендингу из `.env` (`LANDING_PATH`):
```
./venv/Scripts/python.exe src/main.py
```

Интерактивный бот (long polling, работает пока открыт терминал):
```
./venv/Scripts/python.exe src/bot.py
```
Команды: `/report [url] [week|month]`, либо просто прислать ссылку на лендинг.
Бот отвечает только владельцу — chat_id берётся из `TELEGRAM_CHAT_ID` в `.env`.

## Переменные окружения (`.env`, не в git)

- `YANDEX_METRIKA_TOKEN` — OAuth-токен (право чтения статистики)
- `YANDEX_METRIKA_COUNTER_ID` — ID счётчика Метрики
- `TELEGRAM_BOT_TOKEN` — токен бота
- `TELEGRAM_CHAT_ID` — chat_id владельца (whitelist для бота)
- `LANDING_PATH` — дефолтный URL лендинга для `main.py` и `/report` без аргументов

Шаблон без значений — `config/.env.example`. При добавлении новой переменной
окружения — добавлять и туда, и в `src/config.py` (через `os.getenv`).

## Внесение изменений

- Архитектура: один orchestrator + субагенты в `src/agents/`. Каждый агент —
  отдельная зона ответственности (см. список выше). Новую логику добавлять как
  отдельный субагент или метод существующего, не смешивая ответственности.
- `metrika_agent._get_dimension_breakdown()` — общий хелпер для разбивок по
  измерениям Stat API (источники трафика, UTM и т.п.) — для новых разбивок
  использовать его.
- После изменений в агентах — проверить запуском `src/main.py` (или
  `src/bot.py` командой `/report`), что отчёт собирается и приходит в Telegram
  без ошибок.
- Стиль: минимально, без лишних абстракций, без комментариев "что делает код"
  (имена говорят сами за себя).

## Проверка ошибок

1. Синтаксис/импорты: `./venv/Scripts/python.exe -c "import sys; sys.path.insert(0,'src'); import main"`
   (или просто запустить `src/main.py`).
2. Запуск `src/main.py` — должен пройти без traceback и отправить отчёт в
   Telegram. Если упал на `requests.exceptions.HTTPError` — проверить, что
   `.env` заполнен (но НЕ открывать/выводить его содержимое в чат).
3. `__pycache__/` папки — мусор, можно игнорировать/удалять, они в `.gitignore`.

## Подготовка к публикации (git/публичный репозиторий)

- Убедиться, что `.gitignore` содержит `.env`, `venv/`, `__pycache__/`, `*.pyc`.
- НИКОГДА не читать вслух, не логировать и не коммитить содержимое `.env` —
  там реальные токены (Яндекс Метрика, Telegram).
- `config/.env.example` должен содержать все ключи из `.env`, но с пустыми
  значениями — проверять синхронизацию при добавлении новых переменных.
- Перед публикацией прогнать `src/main.py`, убедиться, что всё работает на
  чистом `venv` (`pip install -r requirements.txt`).
