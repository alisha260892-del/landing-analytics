# Landing Analytics — анализатор лендингов 

Инструмент, который автоматически собирает статистику посадочной страницы из
Яндекс Метрики, анализирует её по простым правилам и присылает читаемый отчёт
с диаграммой прямо в Telegram — без захода в интерфейс Метрики.

## Решаемая задача

Маркетологу или владельцу лендинга нужно регулярно проверять, как ведёт себя
страница: сколько визитов, какой процент отказов, откуда приходит трафик.
Обычно для этого нужно заходить в Яндекс Метрику, открывать нужный отчёт,
выбирать период и сегмент по URL. Этот проект автоматизирует весь путь:
данные запрашиваются через API Метрики, проходят автоматическую проверку по
пороговым значениям (отказы, время на сайте) и приходят в виде готового
текстового отчёта с выводами и диаграммой источников трафика.

## Пользователи

- Владельцы лендингов и небольших сайтов, которые хотят быстро видеть
  ключевые метрики без захода в Метрику.
- Маркетологи и фрилансеры, ведущие несколько посадочных страниц и
  отчитывающиеся перед клиентами.
- Любой, кто хочет получать еженедельный/ежемесячный отчёт по странице прямо
  в Telegram.

## Основная функция

Собрать за выбранный период (неделя или месяц) статистику конкретного
лендинга из Яндекс Метрики (Stat API), проанализировать показатели по
пороговым значениям (высокий процент отказов, слишком короткое время на
сайте), сформировать текстовый отчёт с выводами и диаграмму источников
трафика, и отправить всё это в Telegram.

Есть два режима использования:

- **Разовый отчёт** (`src/main.py`) — собирает отчёт по лендингу,
  указанному в `.env` (`LANDING_PATH`), и сразу отправляет его в Telegram.
  Подходит для запуска по расписанию (cron / Планировщик задач).
- **Интерактивный бот** (`src/bot.py`) — запускается и слушает сообщения.
  По команде `/report [url] [week|month]` или просто при отправке ссылки на
  лендинг бот сразу присылает отчёт по этому адресу. Отвечает только
  владельцу (chat id из `.env`).

## Формат результата

Отчёт приходит в Telegram в виде текстового сообщения и изображения:

```
📊 Еженедельный отчет по лендингу zerocoder.ru/v1-claude-code

Визиты: 342
Просмотры: 510
Пользователи: 298
Отказы: 38.4%
Среднее время на сайте: 47 сек

Выводы:
- Показатель отказов в норме: 38.4%
- Время на сайте в норме: 47 сек

Источники трафика:
- direct: 140
- organic: 95
- referral: 62
- ad: 45
```

К сообщению прикладывается диаграмма с разбивкой источников трафика,
построенная с помощью matplotlib.

Пример полного отчёта — `data/example_report.txt`.

## Минимальный функционал (MVP)

- Запрос к Яндекс Метрике (Stat API) с фильтром по URL лендинга и заданным
  периодом (`7daysAgo` / `30daysAgo`).
- Получение базовых метрик: визиты, просмотры, пользователи, процент
  отказов, среднее время на сайте.
- Анализ метрик по пороговым значениям:
  - процент отказов выше 50% — помечается как высокий;
  - среднее время на сайте меньше 30 секунд — помечается как короткое.
- Разбивка трафика по источникам (direct, organic, referral, ad и т.д.).
- Сборка текстового отчёта на русском языке с выводами.
- Построение диаграммы источников трафика.
- Отправка отчёта (текст + картинка) в Telegram через Bot API.
- Telegram-бот с командой `/report [url] [week|month]` и реакцией на
  присланную ссылку, доступный только владельцу.

## Технологии

- **Python 3** — основной язык проекта.
- **Яндекс Метрика Stat API** — источник данных по лендингу (`requests`).
- **python-telegram-bot** — отправка отчётов и работа интерактивного бота.
- **matplotlib** — построение диаграммы источников трафика.
- **python-dotenv** — конфигурация через переменные окружения (`.env`).

## Установка

```
py -m venv venv
./venv/Scripts/python.exe -m pip install -r requirements.txt
```

## Настройка

Скопируй `config/.env.example` в `.env` (в корень проекта) и заполни:

- `YANDEX_METRIKA_TOKEN` — OAuth-токен Яндекс Метрики (право чтения статистики)
- `YANDEX_METRIKA_COUNTER_ID` — ID счётчика
- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather
- `TELEGRAM_CHAT_ID` — ID чата для отправки отчётов
- `LANDING_PATH` — URL лендинга для анализа, например `zerocoder.ru/v1-claude-code`

Файл `.env` не публикуется и не попадает в git.

## Запуск

Разовый отчёт по лендингу из `.env`:

```
./venv/Scripts/python.exe src/main.py
```

Интерактивный бот (команда `/report [лендинг] [week|month]`):

```
./venv/Scripts/python.exe src/bot.py
```

Бот работает, пока открыт терминал (long polling), останавливается Ctrl+C.
